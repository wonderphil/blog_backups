Ruby on Rails

Firstly setting this up on a MacBook Pro running OS Sierra. I like to have a few things installed first but they are not required.
  • iTerm2 - good terminal app for mac - https://www.iterm2.com/
  • Sublime Text 3 - Good Editor - https://www.sublimetext.com/3
  • Growl - For notifications for the OS - http://growl.info/
  • Pow - Pow is a zero-config Rack server for Mac OS X - http://pow.cx/

Prerequisites

Xcode Command Line Tools

Since Mavericks you haven't needed to install Xcode itself to be able to install the command line tools, now you can just run the below command, xcode command line tools is used for build tools and utilities. 
xcode-select --instal

Homebrew

Package manager for Mac, you can use Mac ports but I wouldn't. There isn't much to do install, just follow the steps on the Homebrew site - http://brew.sh/ 
/usr/bin/ruby -e "$(curl -fsSL "https://raw.githubusercontent.com/Homebrew/install/master/install)"

I always run brew doctor to make sure everything is installed correctly. The only other thing I do is an update brew update

Git and Github

Now everyone should be using some sort of source code, so before we go to much further, lets install Git and connect to your Github account. Git is simple, just do a brew install
brew install git

I also Like to install Git completion, which is helpful
brew install bash-completion

For Git completion, you also need to add the following to your bash profile file ~/.bash_profile 
if [ -f $(brew --prefix)/etc/bash_completion ]; then
    . $(brew --prefix)/etc/bash_completion
  fi

Now we configure Git locally with your name and email address: 
git config --global user.name "Your Name Here"
git config --global user.email "your_email@example.com"

Next part is to setup Github ssh keys, now if you don't have Github account, then go get one! To setup ssh keys, git hub has a nice little page, so I won't go through it, just follow the steps at https://help.github.com/articles/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent/
The one thing I would do is make sure you test the connection: 
ssh -T git@github.com

Ruby with RBENV

Ruby is already installed with Mac OS, but its better to install it with a Ruby version manager like RVM or RBENV. RBENV is simpler and more light and so I prefer it over RVM. To install RBENV run the following: 
brew install rbenv ruby-build
git clone https://github.com/sstephenson/rbenv-gem-rehash.git ~/.rbenv/plugins/rbenv-gem-rehash

Now you need to initialize rbenv by adding the following line to you bash profile ~/.bash_profile 
export PATH="$HOME/.rbenv/bin:$PATH"
 eval "$(rbenv init -)"

Now you should be ready for the install of Ruby by simply doing a rbenv install and the version number. You might want to see what version is latest by using the rbenv install -l 
rbenv install 2.3.1
rbenv global 2.3.1

Rails

Last part is installing rails, now this can take some time do don't stress if it seems like nothing is happening. 
gem install rails

Now test everything is working by creating your first application: 
rails new first_app_name

Once the app has finished setting up create the symlink in the Pow folder if you installed Pow: 
cd ~/.pow
ln -s /path/to/app

Then in you browser open the app using "app_name.dev"
You should now see the Rails Landing page.

If you have an error then work back through the above and see if you have missed anything, then next best google the error. If you post the error below then I will try and help where I can.

Installing Postgresql DB - Optional


Now some might call this over kill and this is optional, but I like to install the same DB in my dev environment as whats in my production evironment. So here I will install Postgresql. In recent times Postgresql has released a Postgres.app, which makes it nice and easy to install the server on your mac - http://postgresapp.com/
Now because I like to use the command line tools as well I just add the following line to my bash profile ~/.bash_profile 
export PATH=$PATH:/Applications/Postgres.app/Contents/Versions/latest/bin
