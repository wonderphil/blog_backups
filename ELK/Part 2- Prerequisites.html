Prerequisites

Hardware:

Server:

2gb ram and 2 cores.

Notes:
Elasticsearch
CPU, as this is just a PoC so I can test what disk requirements are going to be like, I have just setup a VM with 2 cores.  To allow better concurrency the more cores the better.

Elasticsearch loves it memory, just like old people with little memory it will come to a stop and take forever to do anything. Elastic says 64GB is on the dollar. Again as this is just a test that I am running on VM's off my desktop I am just running with 2GB, that plus I am not expecting to do huge queries, sorts or aggregations.

Hard drives, this is the all important part of a storage service. I haven't found anywhere to give a good indication on disk sizing, thus why I am testing before going to production.  Now if you are lucky enough to be in a cloud environment where adding disks isn't a problem, then space shouldn't be an issue.

The issue is always going to be I/O. So the faster the disk the better, SSD the best! Elastic does say through try not to use network-attached (NAS) storage, not to be confused with SAN's

Software:

OS:

Ubuntu 14.04 with root access

SSH:

if like me you just have Ubuntu installed as basic as it comes then you probably need to install OpenSSH Server.

sudo apt-get install openssh-server

Java:

First we need to add Oracle Java PPA to apt, then update apt, lastly install Java
sudo add-apt-repository -y ppa:webupd8team/java
sudo apt-get update
sudo apt-get -y install oracle-java8-installer
