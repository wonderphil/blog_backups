Mobile Testing

Requirements:

Let's start with setting up the mobile device.  I am using a Samsung Galaxy S5(sm-g900f) with android version 6.0.1
If you don't know these details have a look in the phone setting -> about device.

This part took me a little bit to get to work, so don't be surprised if it takes some time to figure out.

USB Debugging on Phone

On your Android device, open up Settings, find the Developer options section, and enable USB debugging.  Now on my phone I had to tap Build Number 7 times to enable the the developer menu.

USB Drivers

To be able to connect and test against your mobile device you need to install the drivers, if you go to the https://developer.android.com/tools/extras/oem-usb.html this page has a little table with all the different drive download pages for all the phone.  
Find your driver, download and install, then plug your phone into your computer and make the drivers are picked up.  This can be confirmed by looking in the device manager.

Android SDK Device Debugger

From the SDK tools folder there is a Batch file called monitor, make sure the phone isn't plugged in, start the batch file
C:\Users\Philip\AppData\Local\Android\sdk\tools\monitor.bat


Now that the app is open, unlock your phone and connect via USB.  You should see on the phone a request to allow debugging from the computer you just connected to.

Allow and you should see the app have the phone details.


If you have been able to get that going, then you're ready to start testing!


Finding Elements on Android Web

For this part you have to make sure your sign into chrome on both the desktop and on the phone!
No that you can see the phone with the Android Device Manager, open up Chrome and the Developer tools, from there open up inspect devices...


From here you should see you you phone.  if you open it you will see it list any open browser sessions you have open on your phone.
Here you can open and close any new or old browser sessions.

By click inspect and having chrome open on the phone, you will be able to inspect the page just like you do for a desktop web site:


The important note here is the two icons in the red, the first icon you need to click so you can select what inspect, the second Icon just hides/unhides the display of the phone on the desktop.
