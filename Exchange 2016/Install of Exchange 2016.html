Install of Exchange 2016


Intro


So with the release of Exchange 2016 RTM in October 2015, I have decided to do a quick step by step guide on install and configure.  Basic install in going to happen on a Windows 2012 R2 Server in a basic domain environment, see figure below.

This guide isn't going to work through any upgrade process, its a fresh install and if you need to upgrade I suggest google sorry.

The first thing I will explain through unlike other versions of Exchange, 2016 has only two roles Mailbox and edge Transport.



Prerequisites

  1. Functional Domain level at least Windows Server 2008 or higher
  2. At least one Global catalog server
  3. The server exchange will be installed on must be part of the domain
  4. User installing Exchange must be part of Enterprise Admin, Schema Admin and Domain Admin
  5. Install all required windows features
    Powershell command:
    Add-WindowsFeature RSAT-ADDS, AS-HTTP-Activation, Desktop-Experience, NET-Framework-45-Features, RPC-over-HTTP-proxy, RSAT-Clustering, RSAT-Clustering-CmdInterface, RSAT-Clustering-Mgmt, RSAT-Clustering-PowerShell, Web-Mgmt-Console, WAS-Process-Model, Web-Asp-Net45, Web-Basic-Auth, Web-Client-Auth, Web-Digest-Auth, Web-Dir-Browsing, Web-Dyn-Compression, Web-Http-Errors, Web-Http-Logging, Web-Http-Redirect, Web-Http-Tracing, Web-ISAPI-Ext, Web-ISAPI-Filter, Web-Lgcy-Mgmt-Console, Web-Metabase, Web-Mgmt-Console, Web-Mgmt-Service, Web-Net-Ext45, Web-Request-Monitor, Web-Server, Web-Stat-Compression, Web-Static-Content, Web-Windows-Auth, Web-WMI, Windows-Identity-Foundation
  6. Microsoft Unified Communications Managed API 4.0 must be installed and can be download from here
  7. .NET Framework 4.5.2 must be installed and can be download from here



Next you want to select the role that you are installing, for this step we will select the mailbox role, which also includes the management tools


Now select where to install
